# SpringBoot-Init 项目功能详细介绍文档

> 基于 [程序员鱼皮](https://github.com/liyupi) 的 SpringBoot 项目初始模板  
> 项目坐标：`com:springboot-init:0.0.1-SNAPSHOT`  
> 默认访问地址：`http://localhost:8101/api`  
> 接口文档地址：`http://localhost:8101/api/doc.html`

---

## 一、项目概述

**springboot-init** 是一个基于 Java Spring Boot 的后端项目初始模板，整合了主流框架与常见业务场景的示例代码。项目以「内容社区」为核心业务模型，提供了用户体系、帖子管理、点赞收藏、全文搜索、文件上传、微信集成等完整功能，适合作为快速开发后端 API 的起点。

### 1.1 项目定位

| 维度 | 说明 |
|------|------|
| 类型 | Spring Boot 后端 API 模板 |
| 业务场景 | 内容网站 / 社区论坛 |
| Java 版本 | 1.8 |
| Spring Boot 版本 | 2.7.2 |
| 默认端口 | 8101 |
| 上下文路径 | `/api` |

### 1.2 核心特点

- **开箱即用**：整合 MySQL、Redis、Elasticsearch、腾讯云 COS 等常用中间件
- **规范分层**：Controller → Service → Mapper 标准三层架构
- **权限体系**：基于 Session 的登录态 + 自定义 `@AuthCheck` 注解权限校验
- **代码生成器**：基于 FreeMarker 模板，可快速生成 CRUD 代码
- **多环境配置**：支持 dev / test / prod 环境切换

## 五、业务功能模块详解

### 5.1 用户模块（UserController）

**路由前缀**：`/api/user`

#### 5.1.1 登录认证

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 用户注册 | POST | `/register` | 账号 ≥4 位，密码 ≥8 位，两次密码需一致 |
| 用户登录 | POST | `/login` | 账号密码登录，Session 存储登录态 |
| 微信登录 | GET | `/login/wx_open?code=xxx` | 微信开放平台 OAuth2 登录，自动注册 |
| 用户注销 | POST | `/logout` | 清除 Session 登录态 |
| 获取当前用户 | GET | `/get/login` | 返回脱敏后的 LoginUserVO |

**登录机制**：
- 使用 HttpSession 存储用户信息（Key：`user_login_state`）
- 密码加密：`MD5(SALT + password)`，SALT 固定为 `"yupi"`
- 注册时使用 `synchronized` 防止账号重复
- 微信登录通过 `unionId` 查找或自动创建用户
- 被封号用户（`ban` 角色）禁止微信登录

#### 5.1.2 用户管理（CRUD）

| 接口 | 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|------|
| 创建用户 | POST | `/add` | 管理员 | 默认密码 `12345678` |
| 删除用户 | POST | `/delete` | 管理员 | 逻辑删除 |
| 更新用户 | POST | `/update` | 管理员 | — |
| 获取用户 | GET | `/get?id=` | 管理员 | 返回完整 User 实体 |
| 获取用户 VO | GET | `/get/vo?id=` | 公开 | 脱敏后的 UserVO |
| 分页列表 | POST | `/list/page` | 管理员 | 完整 User 分页 |
| 分页列表 VO | POST | `/list/page/vo` | 公开 | 脱敏分页，pageSize ≤ 20 |
| 更新个人信息 | POST | `/update/my` | 登录用户 | 仅能修改自己的信息 |

**用户角色**：

| 角色值 | 含义 | 权限 |
|--------|------|------|
| user | 普通用户 | 基础操作 |
| admin | 管理员 | 全部管理接口 |
| ban | 被封号 | 禁止登录和操作 |

**查询条件**（UserQueryRequest）：支持按 id、unionId、mpOpenId、userName（模糊）、userProfile（模糊）、userRole 筛选，支持自定义排序字段。

---

### 5.2 帖子模块（PostController）

**路由前缀**：`/api/post`

#### 5.2.1 基础 CRUD

| 接口 | 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|------|
| 创建帖子 | POST | `/add` | 登录用户 | 自动绑定当前用户，初始化点赞/收藏数为 0 |
| 删除帖子 | POST | `/delete` | 本人或管理员 | 逻辑删除 |
| 更新帖子 | POST | `/update` | 管理员 | — |
| 编辑帖子 | POST | `/edit` | 本人或管理员 | 用户侧编辑入口 |
| 获取帖子 VO | GET | `/get/vo?id=` | 公开 | 含作者信息、当前用户点赞/收藏状态 |
| 分页列表 | POST | `/list/page` | 管理员 | 完整 Post 分页 |
| 分页列表 VO | POST | `/list/page/vo` | 公开 | 含作者信息，pageSize ≤ 20 |
| 我的帖子 | POST | `/my/list/page/vo` | 登录用户 | 当前用户创建的帖子 |

**帖子校验规则**：
- 创建时：title、content、tags 均不能为空
- 标题最长 80 字符，内容最长 8192 字符
- tags 以 JSON 数组字符串存储，如 `["Java","Spring"]`

#### 5.2.2 帖子检索

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| MySQL 检索 | POST | `/list/page/vo` | 支持 searchText、title、content、tags、userId 等条件 |
| ES 全文搜索 | POST | `/search/page/vo` | 基于 Elasticsearch 的灵活检索 |

**MySQL 查询能力**（PostQueryRequest）：
- `searchText`：同时模糊匹配 title 和 content
- `tags`：AND 逻辑，必须包含所有指定标签
- `notId`：排除指定 ID
- 支持自定义排序

**ES 搜索能力**（searchFromEs）：
- 关键词搜索：match 匹配 title、description、content
- 标签过滤：AND（tags）/ OR（orTags）
- 过滤 isDelete=0、userId、id、notId
- 默认按相关度评分排序，可指定字段排序
- 搜索结果回查 MySQL 获取最新点赞/收藏数
- ES 中存在但 DB 已删的数据会自动清理

### 5.3 帖子点赞模块（PostThumbController）

**路由前缀**：`/api/post_thumb`

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 点赞/取消点赞 | POST | `/` | 需登录，传 postId |

**业务逻辑**：
- Toggle 模式：已点赞则取消（返回 -1），未点赞则点赞（返回 +1）
- 使用 `@Transactional` + `synchronized(userId)` 保证并发安全
- 通过 AOP 代理调用事务方法（`AopContext.currentProxy()`）
- 同步更新 `post.thumbNum` 计数

### 5.5 文件上传模块（FileController）

**路由前缀**：`/api/file`

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 文件上传 | POST | `/upload` | multipart/form-data |

**请求参数**：
- `file`：文件（multipart）
- `biz`：业务类型（如 `user_avatar`）

**业务类型枚举**（FileUploadBizEnum）：

| 值 | 说明 | 限制 |

**存储规则**：
- 路径格式：`/{biz}/{userId}/{随机8位}-{原文件名}`
- 上传至腾讯云 COS，返回可访问 URL
- 全局单文件大小限制：10MB（application.yml 配置）

## 六、基础设施与通用能力

### 6.1 统一响应格式

所有接口返回 `BaseResponse<T>`：

```json
{
  "code": 0,
  "data": {},
  "message": "ok"
}
```

**错误码体系**（ErrorCode）：

| 错误码 | 含义 |
|--------|------|
| 0 | 成功 |
| 40000 | 请求参数错误 |
| 40100 | 未登录 |
| 40101 | 无权限 |
| 40300 | 禁止访问 |
| 40400 | 数据不存在 |
| 50000 | 系统内部异常 |
| 50001 | 操作失败 |

### 6.2 全局异常处理（GlobalExceptionHandler）

- `BusinessException`：返回业务错误码和自定义消息
- `RuntimeException`：统一返回系统错误

### 6.3 权限校验（AuthInterceptor + @AuthCheck）

```java
@AuthCheck(mustRole = UserConstant.ADMIN_ROLE)  // 需要管理员权限
```

拦截流程：
1. 获取当前登录用户（未登录抛 40100）
2. 校验用户角色是否匹配
3. 被封号用户直接拒绝

### 6.4 请求日志（LogInterceptor）

拦截所有 `controller` 包下的方法，记录：
- 请求 ID（UUID）
- 请求路径、IP、参数
- 响应耗时（毫秒）

### 6.5 跨域配置（CorsConfig）

- 允许所有域名（`allowedOriginPatterns: *`）
- 允许携带 Cookie（`allowCredentials: true`）
- 支持 GET / POST / PUT / DELETE / OPTIONS

### 6.6 Long 精度处理（JsonConfig）

Jackson 序列化时将 `Long` 类型转为字符串，避免前端 JavaScript 大整数精度丢失。

### 6.7 分页支持（MyBatisPlusConfig）

集成 MyBatis Plus 分页插件，支持 MySQL 分页查询。

### 6.8 逻辑删除

MyBatis Plus 全局配置：
- 逻辑删除字段：`isDelete`
- 已删除值：1，未删除值：0
- 适用于 user、post 表；post_thumb、post_favour 为硬删除

## 七、Elasticsearch 集成

### 7.1 索引设计

索引名：`post_v1`（别名 `post`）

| 字段 | ES 类型 | 分词器 |
|------|---------|--------|
| title | text | ik_max_word / ik_smart |
| content | text | ik_max_word / ik_smart |
| tags | keyword | — |
| thumbNum / favourNum | long | — |
| userId | keyword | — |
| createTime / updateTime | date | — |
| isDelete | keyword | — |

Mapping 配置文件：`sql/post_es_mapping.json`

### 7.2 数据同步任务

| 任务类 | 类型 | 触发方式 | 说明 |
|--------|------|----------|------|
| FullSyncPostToEs | 一次性 | CommandLineRunner | 启动时全量同步所有帖子 |
| IncSyncPostToEs | 定时 | @Scheduled(fixedRate=60s) | 每分钟同步近 5 分钟变更的数据 |

> 默认两个任务均未启用（`@Component` 被注释），需手动取消注释开启。

### 7.3 ES 启用步骤

1. 配置 `application.yml` 中的 Elasticsearch 连接信息
2. 取消 `PostEsDTO` 上 `@Document(indexName = "post")` 注释
3. 通过 Kibana 或 API 创建索引（使用 `post_es_mapping.json`）
4. 取消 ES 同步任务的 `@Component` 注释
5. 重启应用

| 生成物 | 模板文件 |

1. 修改 `CodeGenerator.main()` 中的生成参数（packageName、dataName、dataKey 等）

5. 移动到对应包路径，按 `// todo` 提示修改

### 用户接口 `/user`

```
POST   /user/register          用户注册
POST   /user/login             用户登录
GET    /user/login/wx_open     微信开放平台登录
POST   /user/logout            用户注销
```

## 十四、架构设计说明

### 14.1 分层架构

```
┌─────────────────────────────────────────┐
│              Controller 层               │  接收请求、参数校验、权限注解
├─────────────────────────────────────────┤
│               Service 层                 │  业务逻辑、事务控制
├─────────────────────────────────────────┤
│          Mapper / EsDao 层              │  数据访问（MySQL / ES）
├─────────────────────────────────────────┤
│     MySQL / Redis / ES / COS            │  基础设施
└─────────────────────────────────────────┘
```

### 14.2 横切关注点

```
请求 → LogInterceptor（日志） → AuthInterceptor（权限） → Controller
                                                              ↓
                                                    GlobalExceptionHandler
                                                              ↓
                                                         BaseResponse
```

### 14.3 数据对象流转

```
Request DTO → Entity → VO（返回前端）
                ↕
           MySQL / ES
```

- **Entity**：与数据库表一一对应
- **DTO**：接收前端请求参数
- **VO**：返回给前端，可聚合关联数据（如 PostVO 包含 UserVO、hasThumb、hasFavour）
